一次大模型调用，可以理解成：

> **你的软件把一段文字请求发给模型服务商的 API，服务商把文字切成 token，模型计算下一个 token，连续生成，最后把结果返回给你。**

下面用你现在的场景举例：

```
VSCode Claude for Code 插件
→ ccswitch
→ Kimi API / Claude-compatible API
→ 大模型服务器
→ 返回结果
→ VSCode 显示答案
```

------

## 1. API 是什么？

**API** 就是程序之间交流的接口。

你在 VSCode 里输入：

```
帮我重构这个函数
```

VSCode 插件不会自己思考，而是把这句话包装成一个网络请求，发送到某个地址，比如：

```
https://api.xxx.com/v1/chat/completions
```

或者 Claude 风格接口：

```
https://api.xxx.com/v1/messages
```

这个地址就叫 **API endpoint**。

------

## 2. 一次 API 请求通常包含什么？

大概长这样：

```
{
  "model": "kimi-for-coding",
  "messages": [
    {
      "role": "system",
      "content": "你是一个代码助手"
    },
    {
      "role": "user",
      "content": "帮我解释这段代码"
    }
  ],
  "temperature": 0.7,
  "max_tokens": 2000,
  "stream": true
}
```

含义是：

```
model：使用哪个模型
messages：对话内容
system：系统提示词，规定模型身份和规则
user：用户输入
temperature：创造性，越高越发散
max_tokens：最多生成多少内容
stream：是否流式返回
```

------

## 3. Token 是什么？

**Token 是模型处理文字的基本单位。**

它不完全等于“字”或“词”。

例如中文：

```
你好，帮我写代码
```

可能会被切成：

```
你 / 好 / ， / 帮 / 我 / 写 / 代码
```

英文：

```
Hello world
```

可能被切成：

```
Hello / world
```

代码：

```
function hello() {}
```

可能被切成：

```
function / hello / ( / ) / { / }
```

不同模型的切分方式不完全一样。

------

## 4. 为什么 token 重要？

因为大模型计费、上下文长度、输出长度，通常都按 token 算。

一次调用里有两类 token：

```
输入 token：你发给模型的内容
输出 token：模型生成的内容
```

比如：

```
你输入 10,000 token
模型回复 2,000 token
这次调用总共消耗 12,000 token
```

API 服务商通常按：

```
输入 token 单价 + 输出 token 单价
```

收费。

------

## 5. 上下文窗口是什么？

**上下文窗口**就是模型一次能“看见”的最大 token 数。

比如某模型上下文是：

```
128K tokens
```

意思是：

```
系统提示词 + 历史对话 + 代码文件 + 用户问题 + 模型输出
```

加起来不能超过 128K token。

超过之后，插件或中间层通常会：

```
截断旧对话
压缩历史记录
只发送相关代码片段
拒绝请求
```

所以你在 VSCode 里让模型读整个项目时，它不一定真的把所有文件都发给模型，而是会选择相关文件、摘要、当前打开文件等内容。

------

## 6. 模型内部大概怎么生成答案？

它不是像人一样“想一句完整的话再输出”。

而是反复做这个动作：

```
根据已有内容，预测下一个最可能的 token
```

例如你输入：

```
中国的首都是
```

模型可能预测下一个 token 是：

```
北京
```

然后上下文变成：

```
中国的首都是北京
```

再预测下一个 token：

```
。
```

如此循环，直到生成完成。

------

## 7. temperature 是什么？

**temperature 控制随机性。**

低 temperature：

```
更稳定
更保守
更适合代码、事实、格式化输出
```

高 temperature：

```
更有创意
更发散
更适合写作、头脑风暴
```

代码任务一般建议：

```
0.1 - 0.5
```

创意写作可以：

```
0.7 - 1.0
```

------

## 8. stream 流式输出是什么？

你看到模型一个字一个字吐出来，就是 **stream=true**。

非流式：

```
模型生成完整答案 → 一次性返回
```

流式：

```
模型生成一点 → 返回一点 → 前端实时显示
```

VSCode 代码助手一般都用流式输出，因为体验更好。

------

## 9. ccswitch 在中间做了什么？

你现在的链路大概是：

```
VSCode 插件 → ccswitch → Kimi API
```

ccswitch 可能会做这些事：

```
保存 API Key
切换不同模型配置
把 Claude 风格请求转成目标 API 能接受的格式
设置 base_url
指定 model
转发流式响应
处理部分兼容问题
```

所以它像一个“转接头”。

------

## 10. API Key 是什么？

**API Key 就是你的访问凭证。**

类似：

```
sk-xxxxxxxxxxxxxxxx
```

它证明：

```
这个请求是谁发的
是否有权限
该从哪个账号扣费
是否超过限额
```

所以 API Key 不要发给别人，也不要写进公开 GitHub 仓库。

------

## 11. 一次调用的完整流程

完整过程可以这样理解：

```
1. 你在 VSCode 输入问题
2. 插件收集当前上下文，例如文件内容、选中代码、聊天历史
3. 插件组织成 messages
4. ccswitch 读取你配置的 provider
5. ccswitch 加上 API Key、base_url、model
6. 请求发送到 Kimi 的服务器
7. 服务器把输入文本切成 token
8. 模型根据 token 逐步生成输出 token
9. 服务端把 token 流式返回
10. ccswitch 转发给 VSCode
11. VSCode 把结果显示给你
```

------

一句话总结：

> **调用大模型，本质上就是把上下文通过 API 发给模型服务器，模型把文本切成 token 后逐步预测并生成新 token，再把结果返回给你的软件。**
