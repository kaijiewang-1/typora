| 参数          | 作用             |
| ------------- | ---------------- |
| `-o`          | 指定输出文件名   |
| `-c`          | 只编译不链接     |
| `-E`          | 只预处理         |
| `-S`          | 生成汇编文件     |
| `-std=c++17`  | 指定 C++ 标准    |
| `-g`          | 生成调试信息     |
| `-O0/-O2/-O3` | 优化级别         |
| `-Wall`       | 开启常见警告     |
| `-Wextra`     | 开启更多警告     |
| `-Werror`     | 警告当错误       |
| `-I`          | 添加头文件目录   |
| `-L`          | 添加库目录       |
| `-l`          | 链接库           |
| `-D`          | 定义宏           |
| `-U`          | 取消宏           |
| `-v`          | 显示详细编译过程 |
| `-###`        | 只打印将执行命令 |
| `-pthread`    | 线程支持         |
| `-shared`     | 生成动态库       |
| `-fPIC`       | 位置无关代码     |
| `-save-temps` | 保留中间文件     |

从 `.cpp` 到程序跑起来，通常会经过这几步：

**预处理 → 编译 → 汇编 → 链接 → 装入（加载）→ 运行**

你如果想“查看这些步骤”，最常用的是用 `g++` / `gcc` 的命令把中间结果单独生成出来。

------

## 一、整体流程先看懂

假设文件是 `main.cpp`。

### 1. 预处理

处理 `#include`、`#define`、条件编译这些内容，展开头文件，生成预处理后的代码。

### 2. 编译

把预处理后的 C++ 代码翻译成汇编代码。

### 3. 汇编

把汇编代码翻译成机器指令，生成目标文件 `.o`。

### 4. 链接

把目标文件和库文件拼起来，生成最终可执行文件。

### 5. 装入

操作系统把可执行文件加载到内存里，准备运行。

------

## 二、怎么在终端里分别看这些步骤

假设你的源文件：

```
main.cpp
```

------

### 1. 查看预处理结果

```
g++ -E main.cpp -o main.i
```

生成：

```
main.i
```

你可以打开看：

```
cat main.i
```

或者：

```
less main.i
```

这里你会看到：

- `#include <iostream>` 被展开了
- 宏被替换了
- 条件编译被处理了

------

### 2. 查看编译后的汇编代码

```
g++ -S main.cpp -o main.s
```

生成：

```
main.s
```

查看：

```
cat main.s
```

这就是汇编代码。

如果你想更清楚一点，也可以先预处理再编译：

```
g++ -E main.cpp -o main.i
g++ -S main.i -o main.s
```

------

### 3. 查看汇编后的目标文件

```
g++ -c main.cpp -o main.o
```

生成：

```
main.o
```

这是二进制目标文件，不能直接 `cat` 看。

可以用这些命令看内容：

```
file main.o
```

看它是什么文件。

```
nm main.o
```

看符号表，能看到定义了哪些函数、引用了哪些外部符号。

```
objdump -d main.o
```

反汇编，查看机器码对应的汇编。

```
readelf -a main.o
```

查看 ELF 格式的详细信息（Linux 常用）。

------

### 4. 查看链接结果

```
g++ main.o -o main
```

生成可执行文件：

```
main
```

查看文件类型：

```
file main
```

查看它链接了哪些动态库：

```
ldd main
```

查看符号：

```
nm main
```

查看反汇编：

```
objdump -d main
```

------

## 三、如果想直接看到编译器“每一步都在干什么”

可以加 `-v`：

```
g++ -v main.cpp -o main
```

这会把编译器内部调用的步骤显示出来，比如：

- 调用预处理器
- 调用编译器前端
- 调用汇编器 `as`
- 调用链接器 `ld`

这个很适合看“编译和链接到底怎么串起来”。

------

## 四、想只看而不真正执行某一步

可以用 `-###`：

```
g++ -### main.cpp -o main
```

它会**打印将要执行的命令**，但通常不真的执行。

这特别适合学习：

- 编译器背后调用了什么程序
- 链接时加了哪些默认库
- 系统启动文件是怎么参与链接的

------

## 五、装入（加载）步骤怎么查看

“装入”不是 `g++` 做的，而是**操作系统加载器**做的。

程序运行时，OS 会：

- 把可执行文件读进内存
- 映射代码段、数据段
- 加载动态链接库
- 建立栈和堆
- 把程序入口交给 `main` 前的启动代码

### 常见查看方法

#### 1. 看依赖的动态库

```
ldd main
```

#### 2. 看 ELF 文件头和程序头

```
readelf -h main
readelf -l main
```

`-l` 可以看到装入时的段信息。

#### 3. 跟踪程序启动时加载了什么

Linux 下可以用：

```
strace ./main
```

能看到系统调用，比如：

- `execve`
- `open`
- `mmap`
- `read`

这些能帮助你理解程序被装入内存的过程。

#### 4. 查看动态链接器行为

```
LD_DEBUG=libs ./main
```

这会显示动态库加载过程。

------

## 六、一套完整演示命令

```
g++ -E main.cpp -o main.i
g++ -S main.i -o main.s
g++ -c main.s -o main.o
g++ main.o -o main
./main
```

对应关系：

- `main.cpp`：源文件
- `main.i`：预处理后
- `main.s`：汇编代码
- `main.o`：目标文件
- `main`：可执行文件

------

## 七、最常用的“观察命令”总结

### 看编译全过程

```
g++ -v main.cpp -o main
```

### 只看将执行的命令

```
g++ -### main.cpp -o main
```

### 生成预处理文件

```
g++ -E main.cpp -o main.i
```

### 生成汇编文件

```
g++ -S main.cpp -o main.s
```

### 生成目标文件

```
g++ -c main.cpp -o main.o
```

### 看目标文件符号

```
nm main.o
```

### 反汇编

```
objdump -d main.o
```

### 看可执行文件依赖库

```
ldd main
```

### 看装入信息

```
readelf -l main
```

------

## 八、你可以这样理解

一个 `.cpp` 文件不是一下子变成可执行文件的，而是：

```
main.cpp
  ↓ 预处理
main.i
  ↓ 编译
main.s
  ↓ 汇编
main.o
  ↓ 链接
main
  ↓ 装入
运行
```
