## 1. 系统概述

### 1.1 项目背景

随着现代农业技术的快速发展，传统养殖业正面临着向数字化、智能化转型的重大机遇。散养家禽养殖作为一种重要的农业生产形式，在提供优质禽产品、满足消费者对绿色有机食品需求方面具有独特优势。然而，传统的散养家禽养殖方式存在管理效率低、疫病防控困难、环境监测不及时、产品质量追溯体系不完善等问题。

为了有效解决上述问题，提升散养家禽养殖业的现代化管理水平，本项目开发了"散养家禽数智化养殖综合管理平台"。该平台是一套面向养殖企业的综合性智慧农业解决方案，通过集成物联网技术、移动互联网技术和大数据分析技术，实现对养殖场环境、设备、人员和业务的全面智能化管理。

### 1.2 系统定义

本系统是一款基于Android平台的智慧养殖管理移动应用程序，主要服务于中小型养殖企业、养殖专业合作社和家庭农场等养殖主体。系统采用B/S架构模式，移动端作为用户操作终端，通过RESTful API与后端服务器进行数据交互，实现养殖场的日常管理、环境监控、设备运维、工单处理等功能。

系统的核心价值在于将复杂的养殖管理工作简化为直观的移动端操作，使养殖管理人员能够随时随地掌握养殖场状况，及时处理异常情况，提高养殖效率，降低运营成本，最终实现养殖业的数字化、智能化转型升级。

### 1.3 系统目标

本系统的设计目标主要包括以下几个方面：

第一，实现养殖场信息化管理。通过建立电子化的养殖场档案，实现养殖场地、设备、人员等资源的数字化管理，提高管理效率，减少人工记录错误，为企业决策提供数据支撑。

第二，实现环境实时监测。通过部署物联网传感器，实时采集养殖场的温度、湿度、光照、土壤湿度、二氧化碳浓度、风速等环境参数，并通过移动端应用实时展示，帮助管理人员及时了解环境状况，采取相应的调控措施。

第三，实现设备集中管控。通过统一的设备管理平台，实现对摄像头、传感器、控制器等各类智能设备的集中管理，支持设备状态监控、远程控制、设备维护等功能，提高设备利用率，降低设备故障率。

第四，实现工单流程化管理。建立从工单创建、分配、执行到完成的全流程管理机制，支持工单的创建、派发、接单、签到、完成、评价等完整流程，实现工作任务的精细化管理。

第五，实现企业协同办公。支持多用户、多角色的企业组织架构管理，提供企业创建、员工邀请、权限分配等功能，促进养殖企业内部协同工作，提高团队协作效率。

### 1.4 系统特点

本系统具有以下显著特点：

首先是移动优先的设计理念。系统完全基于Android移动平台开发，充分利用移动设备的便携性特点，让管理人员能够随时随地进行业务操作，突破时间和空间的限制。

其次是简洁直观的用户界面。系统采用Material Design设计语言，界面风格简洁美观，操作流程清晰直观，即使是初次使用的用户也能快速上手，降低了培训成本。

第三是实时高效的数据交互。系统采用Retrofit网络框架实现与后端服务的高效通信，支持数据的实时获取和上传，确保信息的时效性和准确性。

第四是灵活的角色权限管理。系统支持管理员、普通用户等多种角色，不同角色拥有不同的功能权限，能够满足养殖企业多层次的管理需求。

第五是良好的用户体验。系统注重细节设计，提供了加载动画、错误提示、确认对话框等交互元素，以及下拉刷新、上拉加载等数据操作方式，提升了用户使用体验。

## 2. 技术架构

### 2.1 总体架构

本系统采用经典的三层架构体系，从上到下依次为表现层、业务逻辑层和数据层。表现层负责用户界面的展示和交互，业务逻辑层处理核心业务规则和数据处理逻辑，数据层负责与后端服务器进行数据交互和本地数据缓存。

在移动端开发技术选型方面，系统采用Kotlin和Java混合编程的方式，充分利用Kotlin语言的现代特性和Java语言的成熟生态。网络通信采用Retrofit框架，结合OkHttp实现高效的HTTP请求处理。数据解析采用Gson库，支持JSON格式数据的序列化和反序列化。

系统的整体技术架构具有良好的扩展性和可维护性，各层之间通过接口进行解耦，便于后续功能扩展和技术升级。

### 2.2 前端技术

#### 2.2.1 开发语言

系统前端采用Kotlin作为主要开发语言，Kotlin是一种运行在JVM上的现代编程语言，它相比Java具有更简洁的语法、更强大的空指针安全检查和更好的函数式编程支持。同时，系统部分代码也使用了Java语言，以充分利用Java生态中成熟的类库和框架。

Kotlin语言的使用使代码更加简洁易读，例如使用数据类（data class）简化模型对象的定义，使用扩展函数增强现有类的功能，使用协程（Coroutine）简化异步编程等。这些特性大大提高了开发效率和代码质量。

#### 2.2.2 用户界面

系统用户界面采用Android原生开发方式，使用XML布局文件定义界面结构，Java代码处理业务逻辑。界面设计遵循Material Design设计规范，采用Material Components库提供的组件，包括MaterialToolbar、MaterialCardView、MaterialButton、BottomNavigationView等，确保界面风格的一致性和美观性。

#### 2.2.3 网络通信

系统网络通信采用Retrofit框架，这是一个类型安全的REST客户端框架，能够将REST API调用转换为Java/Kotlin接口调用。Retrofit框架与OkHttp网络库深度集成，支持请求拦截器、响应拦截器、连接池、缓存等高级功能。

系统实现了AuthInterceptor拦截器，用于在每个请求中自动添加认证令牌（Sa-Token），确保请求的安全性。RetrofitClient单例类封装了Retrofit的创建和配置逻辑，提供了统一的API服务创建方法，简化了网络请求的调用方式。

**RetrofitClient 网络客户端配置：**

```kotlin
// RetrofitClient.kt 网络客户端单例类
object RetrofitClient {
    // 开发环境
    private const val DEV_BASE_URL = "http://localhost:8080/"
    // 生产环境
    private const val PROD_BASE_URL = "http://124.71.97.178:8080/"
    
    private const val BASE_URL = PROD_BASE_URL
    
    private val loggingInterceptor = HttpLoggingInterceptor().apply {
        level = HttpLoggingInterceptor.Level.BODY
    }
    
    private val okHttpClient = OkHttpClient.Builder()
        .addInterceptor(AuthInterceptor())           // 认证拦截器
        .addInterceptor(loggingInterceptor)          // 日志拦截器
        .connectTimeout(30, TimeUnit.SECONDS)        // 连接超时
        .readTimeout(30, TimeUnit.SECONDS)            // 读取超时
        .writeTimeout(30, TimeUnit.SECONDS)          // 写入超时
        .retryOnConnectionFailure(true)              // 自动重连
        .build()
    
    private val retrofit = Retrofit.Builder()
        .baseUrl(BASE_URL)
        .client(okHttpClient)
        .addConverterFactory(GsonConverterFactory.create(GsonBuilder()
            .setDateFormat("yyyy-MM-dd HH:mm:ss")
            .create()))
        .addCallAdapterFactory(CoroutineCallAdapterFactory())
        .build()
    
    fun <T> createService(serviceClass: Class<T>): T {
        return retrofit.create(serviceClass)
    }
}
```

### 2.4 安全机制

系统采用了多重安全机制保障用户数据和应用安全。首先是认证机制，用户登录成功后，后端返回Sa-Token令牌，移动端将令牌存储在本地，后续所有请求都需要在请求头中携带该令牌进行身份验证。

**认证拦截器实现：**

```kotlin
// AuthInterceptor.kt 认证拦截器
class AuthInterceptor : Interceptor {
    override fun intercept(chain: Interceptor.Chain): Response {
        val originalRequest = chain.request()
        
        // 从SharedPreferences获取Token
        val token = AuthManager.getInstance().getToken()
        
        // 构建新的请求，添加Token到请求头
        val newRequest = if (!token.isNullOrEmpty()) {
            originalRequest.newBuilder()
                .header("satoken", token)
                .header("Content-Type", "application/json")
                .build()
        } else {
            originalRequest.newBuilder()
                .header("Content-Type", "application/json")
                .build()
        }
        
        return chain.proceed(newRequest)
    }
}
```

**认证管理器实现：**

```kotlin
// AuthManager.kt 认证管理器单例
class AuthManager private constructor(context: Context) {
    private val prefs = context.getSharedPreferences("auth", Context.MODE_PRIVATE)
    
    companion object {
        @Volatile
        private var instance: AuthManager? = null
        
        fun getInstance(context: Context): AuthManager {
            return instance ?: synchronized(this) {
                instance ?: AuthManager(context.applicationContext).also { instance = it }
            }
        }
    }
    
    fun saveLoginInfo(token: String, userId: Long, enterpriseId: Long) {
        prefs.edit().apply {
            putString("satoken", token)
            putLong("userId", userId)
            putLong("enterpriseId", enterpriseId)
            putLong("loginTime", System.currentTimeMillis())
            apply()
        }
    }
    
    fun getToken(): String? = prefs.getString("satoken", null)
    fun getUserId(): Long = prefs.getLong("userId", 0)
    fun getEnterpriseId(): Long = prefs.getLong("enterpriseId", 0)
    fun getLoginTime(): Long = prefs.getLong("loginTime", 0)
    
    fun isLoggedIn(): Boolean {
        val token = getToken()
        return !TextUtils.isEmpty(token)
    }
    
    fun isTokenExpired(): Boolean {
        val loginTime = getLoginTime()
        val tokenTimeout = 30 * 24 * 60 * 60 * 1000L // 30天

    }

    }
```

### 5.3 数据缓存策略

系统采用本地缓存策略提升数据加载速度和用户体验。对于不经常变化的数据，如企业信息、用户信息等，从后端获取后会缓存到本地，下次访问时优先从本地读取，同时在后台发起请求更新数据。对于频繁变化的数据，如设备状态、环境参数等，每次访问都从后端实时获取，确保数据的时效性。

**本地缓存实现：**

```kotlin
// CacheManager.kt 缓存管理器
class CacheManager(private val context: Context) {
    private val prefs = context.getSharedPreferences("cache", Context.MODE_PRIVATE)
    private val gson = Gson()
    
    companion object {
        private const val DEFAULT_CACHE_TIME = 30 * 60 * 1000L // 30分钟
        
        @Volatile
        private var instance: CacheManager? = null
        
        fun getInstance(context: Context): CacheManager {
            return instance ?: synchronized(this) {
                instance ?: CacheManager(context.applicationContext).also { instance = it }
            }
        }
    }
    
    fun <T> saveCache(key: String, data: T, expireTime: Long = DEFAULT_CACHE_TIME) {
        val cacheData = CacheData(
            data = gson.toJson(data),
            expireTime = System.currentTimeMillis() + expireTime
        )
        prefs.edit().putString(key, gson.toJson(cacheData)).apply()
    }
    
    fun <T> getCache(key: String, clazz: Class<T>): T? {
        val json = prefs.getString(key, null) ?: return null
        
        return try {
            val cacheData = gson.fromJson(json, CacheData::class.java)
            
            if (System.currentTimeMillis() > cacheData.expireTime) {
                prefs.edit().remove(key).apply()
                null
            } else {
                gson.fromJson(cacheData.data, clazz)
            }
        } catch (e: Exception) {
            prefs.edit().remove(key).apply()
            null
        }
    }
    
    fun removeCache(key: String) {
        prefs.edit().remove(key).apply()
    }
    
    fun clearAllCache() {
        prefs.edit().clear().apply()
    }
    
    data class CacheData(
        val data: String,
        val expireTime: Long
    )
}
```

### 5.4 列表加载优化

系统使用RecyclerView组件展示各类列表数据，并采用分页加载策略避免一次加载过多数据导致的性能问题。每个列表页面都有加载更多逻辑，当用户滚动到列表底部时，自动加载下一页数据。列表项使用ViewHolder模式复用视图，减少内存占用。

**分页加载实现：**

```kotlin
// PagingViewModel.kt 分页ViewModel
abstract class PagingViewModel<T> : ViewModel() {
    protected val _dataList = MutableLiveData<List<T>>()
    val dataList: LiveData<List<T>> = _dataList
    
    protected val _isLoading = MutableLiveData<Boolean>()
    val isLoading: LiveData<Boolean> = _isLoading
    
    protected val _hasMore = MutableLiveData<Boolean>()
    val hasMore: LiveData<Boolean> = _hasMore
    
    protected var currentPage = 1
    protected val pageSize = 10
    protected var isLoading = false
    protected var hasMoreData = true
    
    init {
        _dataList.value = emptyList()
        _hasMore.value = true
    }
    
    fun loadFirstPage() {
        if (isLoading) return
        currentPage = 1
        hasMoreData = true
        _hasMore.value = true
        loadData(currentPage, pageSize, isRefresh = true)
    }
    
    fun loadMore() {
        if (isLoading || !hasMoreData) return
        loadData(currentPage + 1, pageSize, isRefresh = false)
    }
    
    fun refresh() {
        loadFirstPage()
    }
    
    private fun loadData(page: Int, size: Int, isRefresh: Boolean) {
        isLoading = true
        _isLoading.value = true
        
        viewModelScope.launch {
            val result = loadPageData(page, size)
            result.onSuccess { data ->
                val currentList = if (isRefresh) emptyList() else (_dataList.value ?: emptyList())
                _dataList.value = currentList + data
                
                hasMoreData = data.size >= size
                _hasMore.value = hasMoreData
                
                if (isRefresh) {
                    currentPage = 1
                }
                if (hasMoreData) {
                    currentPage = page
                }
            }.onFailure {
                // 处理错误
            }
            
            isLoading = false
            _isLoading.value = false
        }
    }
    
    abstract suspend fun loadPageData(page: Int, size: Int): Result<List<T>>
}
```

**RecyclerView Adapter：**

```kotlin
// PagingAdapter.kt 分页适配器
abstract class PagingAdapter<T, VH : RecyclerView.ViewHolder> : RecyclerView.Adapter<VH>() {
    
    private val dataList = mutableListOf<T>()
    
    fun submitList(list: List<T>) {
        dataList.clear()
        dataList.addAll(list)
        notifyDataSetChanged()
    }
    
    fun addList(list: List<T>) {
        val startPosition = dataList.size
        dataList.addAll(list)
        notifyItemRangeInserted(startPosition, list.size)
    }
    
    fun getItem(position: Int): T? {
        return if (position in dataList.indices) dataList[position] else null
    }
    
    override fun getItemCount(): Int = dataList.size
    
    override fun onBindViewHolder(holder: VH, position: Int) {
        onBind(holder, dataList[position])
    }
    
    abstract fun onBind(holder: VH, item: T)
}
```
