## 1. 概述

### 1.1 系统简介

散养家禽数智化养殖综合管理平台是一款基于 Spring Boot 3.5.5 的 RESTful API 服务，提供养殖企业数字化管理的全套后端功能，包括用户管理、企业管理、养殖场管理、设备监控、工单管理等核心业务能力。

### 1.2 技术栈

| 技术         | 版本    | 用途             |
| ------------ | ------- | ---------------- |
| Java         | 17      | 编程语言         |
| Spring Boot  | 3.5.5   | 应用框架         |
| MyBatis-Plus | 3.5.12  | ORM 框架         |
| Sa-Token     | 1.44.0  | 权限认证         |
| MySQL        | -       | 业务数据存储     |
| MongoDB      | -       | 设备时序数据存储 |
| Redis        | -       | 缓存/会话存储    |
| 华为云 OBS   | 3.25.10 | 对象存储         |

### 1.3 基础地址

| 环境     | 地址                      | 说明     |
| -------- | ------------------------- | -------- |
| 开发环境 | http://localhost:8080     | 本地部署 |
| 生产环境 | http://124.71.97.178:8080 | 云服务器 |

所有 API 路径均在此基础上拼接，例如：http://localhost:8080/user/login

### 2.2 启动说明

**步骤 1：配置文件修改**

修改 src/main/resources/application-dev.yaml：

spring:
 datasource:
  url: jdbc:mysql://localhost:3306/stbackendv2?useUnicode=true&characterEncoding=UTF-8
  username: root
  password: 你的数据库密码
 data:
  redis:
   host: localhost
   port: 6379
  mongodb:
   host: localhost
   port: 27017

**步骤 2：Maven 构建**

mvn clean package -DskipTests

**步骤 3：运行服务**

java -jar target/STBackendv2-0.0.1-1215.jar

**步骤 4：验证启动**

访问 http://localhost:8080/actuator/health 返回健康状态即表示启动成功。

### 2.3 认证机制

#### 2.3.1 认证方式

系统采用 **Sa-Token** 框架进行认证，使用 Token 作为身份凭证。

\- **Token 名称**：satoken

\- **Token 有效期**：30 天（2592000 秒）

\- **并发登录**：支持同一账号多地登录

\- **Token 风格**：UUID

#### 2.3.2 登录流程

1. 客户端调用 /user/login 接口
2. 服务端验证手机号和密码
3. 验证成功生成 Token 并返回
4. 客户端在后续请求中携带 Token

#### 2.3.3 Token 使用方式

Token 可通过以下方式传递：

**方式一：请求头（推荐）**

GET /user/info HTTP/1.1
Host: localhost:8080
satoken: 8f7e9d6c-5b4a-3c2d-1e0f-9a8b7c6d5e4f

**方式二：Cookie**

GET /user/info HTTP/1.1
Host: localhost:8080
Cookie: satoken=8f7e9d6c-5b4a-3c2d-1e0f-9a8b7c6d5e4f

**方式三：URL 参数**

GET /user/info?satoken=8f7e9d6c-5b4a-3c2d-1e0f-9a8b7c6d5e4f HTTP/1.1

#### 2.3.4 多租户机制

系统基于 enterprise_id 实现多租户数据隔离：

\- 用户登录后，企业 ID 存储在 Session 中

\- 所有 SQL 查询自动注入 enterprise_id 条件

\- 不同企业的数据完全隔离，无法互相访问

## 3. API 设计规范

### 3.1 统一响应格式

所有接口返回统一的 JSON 格式：

{
 "code": 0,
 "message": "success",
 "data": {}
}

| 字段    | 类型         | 说明               |
| ------- | ------------ | ------------------ |
| code    | number       | 状态码，0 表示成功 |
| message | string       | 响应消息           |
| data    | object/array | 响应数据           |

### 3.2 状态码定义

| code | 说明       | 处理建议                  |
| ---- | ---------- | ------------------------- |
| 0    | 成功       | -                         |
| -1   | 失败       | 查看 message 获取具体原因 |
| 404  | 资源不存在 | 检查请求参数              |
| 401  | 未登录     | 跳转到登录页              |
| 403  | 无权限     | 提示用户权限不足          |

### 3.3 请求头说明

#### 必填请求头

| 请求头       | 说明                       | 示例             |
| ------------ | -------------------------- | ---------------- |
| Content-Type | 请求体类型                 | application/json |
| satoken      | 认证 Token（登录接口除外） | 8f7e9d6c-...     |

#### 可选请求头

| 请求头 | 说明                                  |
| ------ | ------------------------------------- |
| Accept | 期望的响应格式，默认 application/json |

## 4. 接口详解

### 4.1 用户接口

**基础路径：** /user

#### 4.1.1 用户登录

**接口：** POST /user/login

**认证：** 不需要

**请求体：**

{
 "phone": "13800138000",
 "password": "123456"
}

**响应示例：**

{
 "code": 0,
 "message": "登录成功",
 "data": {
  "userId": 1,
  "enterpriseId": 0,
  "satoken": {
   "tokenValue": "8f7e9d6c-5b4a-3c2d-1e0f-9a8b7c6d5e4f",
   "timeout": 2592000
  }
 }
}

**字段说明：**

\- userId：用户 ID

\- enterpriseId：企业 ID，0 表示未加入企业

\- satoken.tokenValue：认证 Token

\- satoken.timeout：Token 有效期（秒）

## 6. 常见问题

### Q1: Token 过期如何处理？

**A:** 当接口返回 401 状态码时，表示 Token 已过期或无效。此时应跳转到登录页，让用户重新登录获取新的 Token。

### Q2: 如何判断用户未加入企业？

**A:** 登录或注册成功后，检查返回的 enterpriseId：

\- 如果为 0，表示用户未加入任何企业

\- 前端可根据此状态引导用户创建企业或输入邀请码加入

### Q3: 多租户数据隔离如何实现？

**A:** 系统基于 MyBatis-Plus 的多租户插件自动实现：

\- 用户登录后，企业 ID 存储在 Session 中

\- 所有 SQL 查询自动添加 enterprise_id = ? 条件

\- 前端无需关心，只需保证用户登录状态即可

### Q4: 文件上传流程是什么？

**A:**

1. 调用 POST /obs/generateSignature 获取签名 URL
2. 使用 PUT 方法上传文件到返回的 signedUrl
3. 上传成功后，将返回的 accessUrl 保存到业务数据中

### Q5: WebSocket 断线如何处理？

**A:** 建议在客户端实现自动重连机制：

function connectWebSocket() {
 const ws = new WebSocket('ws://localhost:8080/ws?satoken=' + token);

 ws.onclose = function() {
  // 3 秒后重连
  setTimeout(connectWebSocket, 3000);
 };

 ws.onerror = function() {
  ws.close();
 };
}

### Q6: 分页参数如何计算？

**A:**

\- 页码 pageNum 从 1 开始

\- 每页数量 pageSize 建议默认 10

\- 总页数 = Math.ceil(total / pageSize)

### Q7: 时间格式是什么？

**A:** 所有时间字段统一使用 yyyy-MM-dd HH:mm:ss 格式，时区为 GMT+8。
